# Multi-Department Agent Visualization System (Action-Driven Agents)

---

## 1. Overview

This system visualizes employees as agents inside departments.
Agents behave based on the **latest activity log (last action)** of the employee.

Core behavior:

* If a log exists → agent performs a **specific animation**
* If no log → agent performs **random idle behavior**

This creates a real-time, human-readable visualization of operations.

---

## 2. Key Concept: Action-Driven Agents

Each agent’s behavior is controlled by:

```json
{
  "last_log": {
    "action_type": "CREATE_QUOTATION",
    "timestamp": "12:01",
    "metadata": {...}
  }
}
```

---

### Behavior Rule

| Condition          | Agent Behavior           |
| ------------------ | ------------------------ |
| Has latest log     | Perform mapped animation |
| No log / empty log | Random idle movement     |

---

## 3. Action → Animation Mapping (CRITICAL)

You need a **mapping layer**. Do NOT hardcode this everywhere or your future self will hate you.

---

### Example Mapping Table

```json
{
  "CREATE_QUOTATION": "typing_at_computer",
  "CREATE_SO": "typing_at_computer",
  "CREATE_PO": "typing_at_computer",
  "STOCK_TRANSFER": "walking_with_box",
  "MEETING": "sitting_meeting_room",
  "BREAK": "idle_relax"
}
```

---

### Animation Rules

* `typing_at_computer` → agent sits at desk + typing loop
* `walking_with_box` → agent moves across area
* `idle_relax` → random idle animations

---

## 5. Behavior Engine

### Core Logic

```python
if agent.last_log exists:
    action = map_log_to_action(agent.last_log.action_type)
    play_animation(action)
else:
    run_random_idle()
```

---

### Important Rule

Agent keeps doing the SAME action until:

* A **new log is received**

No constant switching. No chaos.

# 16. Asset Strategy (Models, Characters, Environment)

---

## 16.1 Overview

This system relies heavily on visual clarity.
All assets must follow a **consistent art style** to maintain immersion and readability.

Mixing different styles (e.g. realistic characters + cartoon furniture) will break the visual experience.

---

## 16.2 Asset Categories

You will need:

### Characters (Agents)

* Office employees
* Sitting / typing animations
* Walking animations
* Idle animations

---

### Environment

* Office floor / tiles
* Walls / rooms
* Doors

---

### Furniture

* Tables / desks
* Chairs
* Computers / monitors
* Meeting tables

---

### Props (Optional but valuable)

* Phones (for call actions)
* Documents / boxes (for logistics actions)
* Coffee machines (idle zones)

---

## 16.3 Asset Sources

### Option 1: Ready-Made Assets (Recommended for MVP)

Sources:

* Kenney.nl (free, consistent style packs)
* itch.io asset marketplace
* OpenGameArt.org
* Unity Asset Store (even if not using Unity)

Pros:

* Fast
* Pre-animated assets available
* Consistent packs exist

Cons:

* Limited customization
* Style may not perfectly match your vision

---

### Option 2: AI-Generated Assets

Tools:

* Midjourney
* DALL·E
* Stable Diffusion

Usage:

* Generate characters, furniture, icons

CRITICAL RULE:

* Use the SAME prompt style for ALL assets

Example prompt style:
"top-down 2D office style, minimal, soft colors, consistent lighting, game asset style"

Pros:

* Fully customizable
* Unique visuals

Cons:

* Hard to maintain consistency
* No built-in animations (you must create separately)

---

### Option 3: Hybrid Approach (BEST BALANCE)

* Use a base asset pack (e.g. Kenney office pack)
* Generate missing elements with AI
* Match colors and proportions manually

---

## 16.4 Style Consistency Rules

All assets must share:

* Same perspective (top-down or isometric)
* Same color palette
* Similar proportions
* Similar lighting and shadow style

---

### DO NOT:

* Mix pixel art with realistic assets
* Mix 2D flat with 3D rendered styles
* Use different camera angles

---

## 16.5 Animation Requirements

Minimum animations per agent:

* Idle
* Walking
* Sitting (typing)
* Optional:

  * Carrying item
  * Phone call

---

## 16.6 Asset Pipeline

1. Choose base style (e.g. top-down office)
2. Collect or generate assets
3. Normalize:

   * size
   * scale
   * colors
4. Convert into spritesheets (if needed)
5. Load into engine (Excalibur / Phaser)

---

## 16.7 Optimization Notes

* Keep assets lightweight
* Avoid large textures
* Reuse sprites (do not duplicate unnecessarily)

---

## 16.8 Key Insight

Assets are not decoration.

They are:

> The language your system uses to communicate business activity visually

Bad assets = confusing system
Good assets = instant understanding
